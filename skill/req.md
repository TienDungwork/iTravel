CHƯƠNG 3: PHÂN TÍCH VÀ THIẾT KẾ HỆ THỐNG
----------------------------------------

3.1 Phân tích thực trạng
- Phân tích thị trường/lĩnh vực liên quan
- Thực trạng hiện tại
- Các vấn đề tồn tại
- Số liệu minh họa hợp lý (gần thời điểm hiện tại)

3.1.2 Phân tích nhu cầu người dùng
- Phân tích theo từng nhóm Actor
- Mô tả khảo sát giả định có tỷ lệ %
- Chỉ ra nhu cầu cốt lõi

3.2 Phân tích yêu cầu hệ thống
- Liệt kê Actor
- Mục tiêu từng Actor (dạng bảng text)
- Use-case chính
- Yêu cầu chức năng
- Yêu cầu phi chức năng
- Ràng buộc hệ thống

4.1 Triển khai hệ thống

4.1.1 Cấu trúc mã nguồn
- Mô tả cấu trúc thư mục Backend
- Mô tả cấu trúc thư mục Frontend

4.1.2 Giao diện chương trình
- Mô tả các giao diện chính
- Mô tả luồng người dùng

4.2 Mục tiêu kiểm thử
- Functional Testing
- UI Testing
- Security Testing
- Performance Testing
- Integration Testing

4.3 Thực hiện kiểm thử
- Tạo bảng test case cho:
  + Đăng nhập
  + CRUD dữ liệu chính
  + Quy trình nghiệp vụ chính
  + Phân quyền
  + API
- Có ID
- Có dữ liệu kiểm thử
- Có kết quả mong đợi
- Có trạng thái
